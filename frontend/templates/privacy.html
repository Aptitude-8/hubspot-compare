Understanding how your data is handled in the HubSpot Property Comparison Tool

üîë Token Storage & Security

What We Store

  • HubSpot Private App Tokens - Temporarily stored in server memory
  • Portal Names - The friendly names you assign to each portal
  • Session ID - A unique identifier for your comparison session
  • Cached Property Data - Property schemas and metadata (not actual contact/company data)

What We DON'T Store

  • No Contact Data - We never access or store actual contact, company, or deal records
  • No Personal Information - No names, emails, or other personal data
  • No Persistent Storage - Nothing is saved to disk or databases
  • No Logs - Tokens are never written to log files

‚è∞ Data Retention & Cleanup

1

Session Creation

When you validate tokens, a session is created with a unique ID. Your tokens are stored in server memory only.

2

Active Usage (1 Hour)

Your session remains active for 1 hour. During this time, property data is cached to improve performance.

3

Session Expiration

After 1 hour of inactivity, your session automatically expires and all data is permanently deleted.

4

Automatic Cleanup

Every 5 minutes, the server runs a cleanup process to remove expired sessions and free memory.

ÔøΩÔøΩ Caching Details

What Gets Cached

  • Property Schemas - Cached for 15 minutes to improve performance
  • Object Types - Cached to reduce API calls
  • Association Configurations - Cached for comparison efficiency

Cache Management

  • Automatic Refresh - You can manually refresh cached data anytime
  • Session-Based - Cache is tied to your session and deleted when session expires
  • Memory-Only - Cache exists only in server memory, not on disk
